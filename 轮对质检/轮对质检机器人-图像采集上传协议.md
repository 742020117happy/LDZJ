> 场景: 轮对从车体拆下后单独检修
> Robot_App: 机器人本体独立调度软件, 收集各模块信息, 解析协议, 逻辑判断, 协同各模块完成轮对质检

- 通过 TL-XCPE3000DG (WiFi6 客户端) 连接现场网络

## 二、WEB工作站通信协议 (重点: Robot_App)

### 2.1 连接

TCP Socket, JSON 格式, Robot_App 为 Server 端, WEB工作站为 Client 端。
连接后 WEB工作站首先下发巡检任务, Robot_App 自检后应答 READY。
所有往来消息均携带 `Checksum` 字段, Robot_App 在响应中原样回传该值, 用于请求-应答配对校验。

### 2.2 下发巡检任务 (WEB协商版)

> **2026-07-16 协商更新**: 原单任务格式改为 `tasks[]` 数组，每个轮对独立一个 `taskId`。

**与旧协议 (第3版) 关键差异**:

| 对比项 | 第3版原协议 | WEB协商版 |
|---|---|---|
| 任务编号 | 顶层一个 `taskId` | `tasks[].taskId` 每轮对独立 |
| 轮对编号 | 顶层一个 `wheelsetNo` | `tasks[].wheelsetNo` |
| 车轴编号 | 顶层一个 `axleNo` | `tasks[].axleNo` |
| 摆放位置 | 顶层 `wheelsetPositions[]` 数组 | `tasks[].wheelsetPosition` 每任务一个 |
| 执行顺序 | `wheelsetPositions` 顺序 | `tasks` 数组顺序 |
| 报警/复核/统计 | 多位置共用一任务 | 各 `taskId` 独立 |
| 质量报告 | 可能混在同一任务 | WEB 按各自 `taskId` 独立生成 |

### 2.3 控制指令字段

`command` 取值表:

| 命令 | 发送方 | 说明 |
|------|------|------|
| `START_INSPECTION` | WEB工作站→Robot_App | 下发任务数组，按 `tasks` 顺序依次执行 |
| `Cancel_Task` | WEB工作站→Robot_App | 取消当前任务 (仅 IDLE/READY/COMPLETED/ERROR 允许) |
| `Pause` | WEB工作站→Robot_App | 暂停采集 (仅 COLLECTING 允许) |
| `Resume` | WEB工作站→Robot_App | 暂停恢复 (仅 PAUSED 允许) |
| `Emergency_Stop` | WEB工作站→Robot_App | 软急停 (所有状态均可) |
| `Return_Home` | WEB工作站→Robot_App | 回原点 (仅 IDLE/READY/COMPLETED/ERROR 允许) |
| `Read_Ready` | WEB工作站→Robot_App | 轮询全状态, 响应不触发逻辑 |

`START_INSPECTION` 参数字段:

| 字段 | 类型 | 示例 | 说明 |
|------|------|------|------|
| `command` | string | START_INSPECTION | 固定命令字 |
| `wheelsetCount` | int | 2 | 任务数量, **必须等于 `tasks.length`** |
| `tasks[]` | array | — | 独立任务数组，按数组顺序执行 |
| `tasks[].taskId` | string | 202607161530RD3A001 | 任务编号, 每轮对唯一且独立 |
| `tasks[].axleType` | string | RD3A | 轴型 (对应 `InspectionPlan.json`) |
| `tasks[].wheelsetNo` | string | LZ20260716001 | 轮对编号 |
| `tasks[].axleNo` | string | CP20260716001 | 车轴编号 |
| `tasks[].sendUnit` | string | XX车辆段 | 送检单位 |
| `tasks[].startTime` | string | 202607161530 | WEB 指令下发时间 `yyyyMMddHHmm` |
| `tasks[].repairLevel` | string | 三级修 | 修程等级 |
| `tasks[].wheelsetPosition.posId` | int | 1 | 摆放位置编号 |
| `tasks[].wheelsetPosition.mapPointName` | string | wpos_slot_A1 | Magic 地图点名称 |

> **协商变更**: `currentWheelset` → `currentTaskIndex`, `wheelsetCount` → `taskCount`。`taskId` 始终为当前正在执行的任务编号。`chassis`、`arm`、`camera`、`server`、`recentAlarms` 和 `Checksum` 保持不变。

### 2.5 状态掩码与命令准入

| 命令 | 允许状态 | 掩码组合 |
|------|---------|---------|
| `Cancel_Task` | IDLE / READY / COMPLETED / ERROR | (1) | READY(2) | COMPLETED(16) | ERROR(32) |
| `Pause` | COLLECTING | 采集(4) |
| `Resume` | PAUSED | 暂停(8) |
| `Emergency_Stop` | 所有状态 | 全掩码 |
| `Return_Home` | IDLE / READY / COMPLETED / ERROR | 空闲(1) | 就绪(2) | 完成(16) | 错误(32) |

### 2.6 状态掩码实现

Robot_App 使用 `workState` 与位掩码进行命令准入判断:

```cpp
enum StateMask {
    SM_IDLE       = 1 << 0,  // 空闲
    SM_READY      = 1 << 1,  // 就绪
    SM_COLLECTING = 1 << 2,  // 采集
    SM_PAUSED     = 1 << 3,  // 暂停
    SM_COMPLETED  = 1 << 4,  // 完成
    SM_ERROR      = 1 << 5,  // 错误
};
int st = g_Work_DB.workState;
if (!((1 << st) & allowedMask))
    Reply_Error(cmd, "当前[" + name[st] + "]状态不允许" + cmd);
```

## 四、第1层: 机械臂(CGXi) ↔ 上位机(Robot_App)

### 4.1 连接方式

CGXi (IP 192.168.6.6) 通过 Socket TCP (ID=8) 连接 Robot_App (192.168.6.35:9000)。

**Robot_App 端**: `c_Robot_Server` (别名 `QTcpServer`) 监听 0.0.0.0:9000, 收到数据后触发 `Read_Done` 信号, 经 `On_CGXi_Data()` 缓冲及 `!` 切割, 交由 `On_CGXi_Message()` 解析。

**CGXi 端**: 使用 LUA 脚本中的 Socket API 进行连接和通信。

### 4.2 CGXi LUA Socket API

| 函数 | 说明 |
|------|------|
| `socket_connect("tcp", ip, id, port)` | 建立TCP连接, id为连接句柄 |
| `socket_disconnect(id)` | 断开连接 |
| `socket_send_string(id, str)` | 发送字符串 |
| `socket_read_string(id, timeout_ms)` | 读取字符串, 超时返回nil |
| `get_socket_connect_status(id)` | 0=已连接, 1=失败, 3=无效句柄 |

### 4.3 CGXi → Robot_App

#### START

CGXi 轨迹脚本开始执行时, 待各工位就绪, 发送 `START` 通知 Robot_App 进入采集状态:

```
START&!
```

**Robot_App 后续动作**:
1. `workState = 2`, `m_LastGain = -1`
2. 检查 WEB 是否已下发完整任务参数 (`sendUnit`, `startTime`, `wheelsetNo`, `axleNo`, `taskId`)
3. 若参数完整则合成第2层相机 START 指令, 发送至相机采集软件

#### PHOTO

CGXi 到达示教点位后, 停留并发送 PHOTO 指令, 包含增益值和部件参数:

```
CHANG&GAIN&{增益值}&PHOTO&{程序索引}-{部件编号1}-{部件编号2}-{点位编号}!
```

| 参数 | 示例 | 说明 |
|------|------|------|
| 增益值 | 3 | 相机增益 (1~100) |
| 程序索引 | 02 | CGXi 当前运行的程序索引编号 |
| 部件编号1 | TCLD | 部件分类代码 |
| 部件编号2 | 02 | 部件细分编号 |
| 点位编号 | 03 | 当前拍摄点位序号 (同一工位内) |

**完整示例**:

```
CHANG&GAIN&3&PHOTO&02-TCLD-02-03!
```

> **注意**: 第1层拼写 `CHANG` (无E), 第2层拼写 `CHANGE` (有E). Robot_App 负责在两层之间转换。

**Robot_App 后续动作**:
1. 去除 `!`, 按 `&` 和 `-` 分割提取5个参数
2. 校验: GAIN值范围(0-100), PHOTO参数长度(4段)
3. `m_LastGain` 优化: 若 `currentGain` 与上次不同, 先发送 `CHANGE&GAIN` 指令
4. 综合 WEB 任务 + CGXi 参数, 通过 `Build_Photo_Filename()` 合成完整文件名
5. 发送第2层 `CHANGE&GAIN&{gain}` 和 `PHOTO&{wsNo}{axNo}&{完整文件名}`

#### FINISH

当前工位所有 PHOTO 完成后, 轨迹脚本通知 Robot_App 工位结束:

```
PHOTO&FINISH!
```

**Robot_App 后续动作**: 提取 `PHOTO&FINISH` 发送给相机采集软件, `currentPos++`。

### 4.4 Robot_App → CGXi

```
<COMPLETE><T1>!
```

**Robot_App 触发时机**: 收到相机反馈 `PHOTO&COMPLETE` 后, 转发给 CGXi, 通知相机已完成当前 PHOTO 拍摄, 可以继续下一组动作。

由于反馈可能包含文件名, `COMPLETE` 判断采用:
- 优先精确匹配 `"PHOTO&COMPLETE"`
- 兜底前缀匹配 `msg.startsWith("PHOTO&COMPLETE")` (兼容相机返回带有文件名的变体)

### 4.5 CGXi 脚本实现细节

脚本以 `.crscript` 文件形式部署在 CGXi 控制器中, 通过 Modbus 寄存器 88/89 触发执行。这些脚本与 Robot_App 通过 Socket TCP 交互完成采集流程。

**通用辅助函数**: 所有脚本在发送前调用 `flush_recv_buffer()` 清除 socket 残留数据。

```lua
function flush_recv_buffer()
    while true do
        local ok, data = pcall(socket_read_string, 8, 50)
        if not ok or data == nil or data == "" then break end
        print("Flushed residual: " .. data)
    end
end
```

#### START.crscript 连接 + 发送

```lua
-- 1. 断开旧连接
local status = get_socket_connect_status(8)
if status == 0 then socket_disconnect(8) end
wait(500)

-- 2. 建立TCP连接
socket_connect("tcp", "192.168.6.35", 8, 9000)

-- 3. 等待连接就绪 (20次×500ms = 10s超时)
local retry = 20
local connected = false
while retry > 0 do
    local st = get_socket_connect_status(8)
    if st == 0 then connected = true; break
    elseif st == 1 or st == 3 then break end
    retry = retry - 1; wait(500)
end

-- 4. 清空缓冲后发送 START&!
if connected then
    flush_recv_buffer()
    socket_send_string(8, "START&!")
end
```

#### PHOTO.crscript 拍照 + 等待应答

```lua
function PHOTO(gain, program_index, part_1, part_2, point_id)
    -- 转换参数为字符串
    gain = tostring(gain or "1")
    program_index = tostring(program_index or "01")
    part_1 = tostring(part_1 or "TCLD")
    part_2 = tostring(part_2 or "01")
    point_id = tostring(point_id or "01")

    if get_socket_connect_status(8) ~= 0 then
        print("Socket 8 is not connected")
        return
    end

    -- 构造发送字符串
    local send_str = "CHANG&GAIN&" .. gain .. "&PHOTO&"
        .. program_index .. "-" .. part_1 .. "-" .. part_2 .. "-" .. point_id .. "!"
    flush_recv_buffer()
    socket_send_string(8, send_str)

    -- 循环等待 Robot_App 的 COMPLETE 应答
    -- 采用 buffer 累积 + string.find 应对 TCP 粘包
    local buffer = ""
    while true do
        if get_socket_connect_status(8) ~= 0 then
            print("Socket disconnect while waiting")
            break
        end
        local ok, recv_str = pcall(socket_read_string, 8, 2000)
        if ok and recv_str ~= nil and recv_str ~= "" then
            buffer = buffer .. recv_str
            if string.find(buffer, "<COMPLETE><T1>!") then
                print("Receive COMPLETE signal")
                break
            end
        end
        wait(10)
    end
end

-- 调用示例 (实际值由轨迹程序传递)
PHOTO("3", "02", "TCLD", "02", "03")
```

#### FINISH.crscript 结束工位

```lua
if get_socket_connect_status(8) == 0 then
    flush_recv_buffer()
    socket_send_string(8, "PHOTO&FINISH!")
end
```

#### 一体化脚本 wheel_photo.lua

将连接、START、PHOTO、FINISH 封装为可复用库, 供轨迹脚本调用:

```lua
function connect_server() ... end      -- 连接 + 重试
function send_start() ... end          -- flush + START&!
function send_photo(...) ... end       -- flush + CHANG&PHOTO + 等待COMPLETE
function send_finish() ... end         -- flush + PHOTO&FINISH!

    if not send_start() then socket_disconnect(8); return end

    -- 工位点位列表: {gain, prog, part1, part2, point}
    local points = {
        { 3, 10, "TCLD", "01", "01" },
        { 3, 10, "TCLD", "01", "02" },
        ...
    }
    for i, pt in ipairs(points) do
        if not send_photo(pt[1], pt[2], pt[3], pt[4], pt[5]) then
            break  -- 失败时继续发送FINISH
        end
    end
    send_finish()
    socket_disconnect(8)
end
main()
```

## 七、Robot_App 关键实现

### 7.1 粘包 TCP 缓冲区处理

```cpp
// m_CGXi_Buffer 累积数据, 按 ! 切割完整消息
void c_Lczh_Remote::On_CGXi_Data(QByteArray data)
{
    m_CGXi_Buffer.append(data);
    int pos;
    while ((pos = m_CGXi_Buffer.indexOf('!')) >= 0) {
        QByteArray msg = m_CGXi_Buffer.left(pos + 1);  // 包含 !
        m_CGXi_Buffer.remove(0, pos + 1);
        On_CGXi_Message(msg);
    }
}
```

### 7.2 消息路由实现

```cpp
// ========== 收到 CGXi → Robot_App (TCP 9000) ==========
void c_Lczh_Remote::On_CGXi_Message(QByteArray buffer)
{
    if (buffer.isEmpty()) return;
    QString msg = QString::fromUtf8(buffer).trimmed();
    if (msg.isEmpty()) return;

    auto &work = c_Variable::getInstance().g_Work_DB;

    if (msg == "START&!") {
        work.workState = 2;
        m_LastGain = -1;                                  // 重置增益优化

        if (work.sendUnit.isEmpty() || work.startTime.isEmpty()
            || work.wheelsetNo.isEmpty() || work.axleNo.isEmpty()
            || work.taskId.isEmpty()) {
            emit Status("CGXi 发来 START 但 WEB 任务参数尚未就绪");
            return;
        }

        // 合成发送至相机的 START 指令
        QString cmd = QString("START&%1&%2&%3&%4%5&%6")
            .arg(work.sendUnit)
            .arg(work.startTime.left(8))              // yyyymmdd
            .arg(work.startTime.right(4))             // HHMM
            .arg(work.wheelsetNo)
            .arg(work.axleNo)
            .arg(work.taskId);
        emit Signal_Remote_Write(cameraIP, cameraPort, cmd);
    }
    else if (msg.startsWith("CHANG&GAIN&")) {       // 注意: 无E
        // 基本校验
        if (!msg.endsWith('!')) return;
        int photoIdx = msg.indexOf("&PHOTO&");
        if (photoIdx < 0) return;

        QString s = msg.left(msg.length() - 1);      // 去除 !
        QStringList parts = s.split('&');
        if (parts.size() < 5) return;

        // 提取增益值
        int gain = parts[2].toInt(&ok);
        if (!ok || gain < 0 || gain > 100) return;
        work.currentGain = gain;

        // 解析PHOTO部件参数
        QStringList photo = parts[4].split('-');
        work.currentProg = photo[0].toInt();
        work.currentPart1 = photo[1];
        work.currentPart2 = photo[2];
        work.currentPointStr = photo[3];

        // m_LastGain 优化: 仅当增益变化时发送 CHANGE&GAIN
        if (work.currentGain != m_LastGain) {
            m_LastGain = work.currentGain;
            emit Signal_Remote_Write(cameraIP, cameraPort,
                QString("CHANGE&GAIN&%1").arg(work.currentGain));  // 注意: 有E
        }

        // 合成完整文件名后发送 PHOTO 指令
        m_LastPhotoName = Build_Photo_Filename();
        emit Signal_Remote_Write(cameraIP, cameraPort,
            QString("PHOTO&%1%2&%3")
                .arg(work.wheelsetNo, work.axleNo, m_LastPhotoName));
    }
    else if (msg == "PHOTO&FINISH!") {
        emit Signal_Remote_Write(cameraIP, cameraPort, "PHOTO&FINISH");
        work.currentPos++;
    }
    else {
        emit Status("CGXi 收到未识别的消息 -> " + msg);
    }
}
```

#### TCP 粘包处理

基于 `{ }` 括号深度计数处理半包/粘包, JSON 必须在同一次 `Read_Done` 中被完整解析:

```cpp
void c_Server_Remote::Read_Done(QByteArray data)
{
    m_RecvBuffer.append(data);
    int braceDepth = 0, start = -1, consumed = 0;
    for (int i = 0; i < m_RecvBuffer.size(); ++i) {
        char c = m_RecvBuffer[i];
        if (c == '{') {
            if (braceDepth == 0) start = i;
            braceDepth++;
        } else if (c == '}') {
            braceDepth--;
            if (braceDepth == 0 && start >= 0) {
                QByteArray piece = m_RecvBuffer.mid(start, i - start + 1);
                QJsonDocument doc = QJsonDocument::fromJson(piece);
                if (!doc.isNull() && doc.isObject())
                    Process_Command(doc.object());
                consumed = i + 1; start = -1;
            }
        }
    }
    m_RecvBuffer = m_RecvBuffer.mid(consumed);
}
```

#### Process_Command 路由

1. 从请求 JSON 中提取 `Checksum` 存入 `m_LastChecksum`, 后续所有响应通过 `Write_Json_With_Checksum()` 自动回传
2. 双字段查找: 先查 `Cmd_Name`, 再查 `command`
3. `START_INSPECTION` 调用 `Parse_Start_Inspection()` 解析 `tasks[]` 数组, 校验 `wheelsetCount==tasks.length`, 依次填充 `g_Work_DB` 并逐任务 `emit Start_Cmd`
4. `Read_Ready` 调用 `Build_System_Status()` 返回 7 段 JSON 全状态
5. 控制命令 (`Cancel_Task` / `Pause` / `Resume` / `Emergency_Stop` / `Return_Home`) 先经 `Check_State()` 校验状态掩码, 通过后 `emit` 对应信号

#### Build_System_Status 输出

7 段 JSON 分别来自不同数据源:

| 段 | 数据源 | 包含字段 |
|----|--------|---------|
| `task` | `g_Work_DB` | taskId, workState, currentTaskIndex, taskCount, currentPos, totalImages |
| `chassis` | `g_Magic.ReadData` | connected, localized, battery, position(x/y/angle) |
| `arm` | `g_CGXi.data` | connected, mode, programStatus, moving, errorCode, tcpPose |
| `camera` | `g_Prec_Scan_120` | remoteConnected, monitorConnected, tranConnected, tranListen |
| `server` | `g_Server` | clientConnected, listening, clientCount |
| `recentAlarms` | `g_Worry_List` | 最近5条报警记录 |

### 11.2 Robot_App 集成架构

```
Robot_App_Widget (主线程)

       └── (内部) m_Voice_Remote_Thread → ModeBusRTU_Master (QModbusRtuSerialMaster)
```

**c_Voice_Remote 类** (继承 c_Object):
- 主线程创建并 moveToThread 到 `m_Voice_Remote_Thread`, 内部 `Init()` 再创建 `c_ModeBusRTU_Master` 并 moveToThread 到第二个线程

- 写操作通过请求队列 (`m_Set_HoldingRegisters[2]`) + `Write_HoldingRegisters_Action()` 统一分派
- 全局状态通过 `c_Variable::getInstance().g_Voice` (s_Voice_DB 结构体) 对外暴露, 主线程 `Voice_Scan()` 每 10ms 读取

### 11.6 c_Voice_Remote 接口

**公共槽函数** (可在主线程通过信号槽跨线程调用):

```cpp
// 生命周期
void Init();                // 初始化 Modbus RTU Master + 创建内部线程 + 绑定信号槽 + 连接
void Connect();             // 从 Communicate_DB.json 读取串口参数, 发起 Modbus RTU 连接
void Connect_Loop();        // 断线重连: 未连接时每 3 秒重试

// 音频播报 (25 条命名接口 + 通用接口)
void Play(int index);       // 通用接口: 0=停止, 1-25=播放对应音频
void Play_PA001(); ... void Play_PA025();  // 对应索引 1-25
void Stop();                // 停止播放 (等价于 Play(0))
void SetVolume(int vol);    // 设置音量 1=低 / 2=中 / 3=高, 同时持久化到 Communicate_DB.json
```

**跨线程信号** (转发给内部的 `c_ModeBusRTU_Master`):

```cpp
void Signal_Connect_Device(QString, int, int, int, int, int);    // 发起串口连接
void Signal_Disconnect_Device();                                   // 断开串口连接
void Signal_Write_HoldingRegisters(int, int, s_ModBus_DB, int);   // 写入保持寄存器
void Signal_Read_HoldingRegisters(int, int, int);                  // 读取保持寄存器
```

**全局状态结构体 `s_Voice_DB`** (通过 `c_Variable::getInstance().g_Voice` 访问):

| 字段 | 类型 | 说明 |
|------|------|------|
| `is_connected` | bool | Modbus RTU 连接状态 |
| `Ready` | bool | 连接成功且音量已设 |
| `Volume` | int | 当前音量等级 1-低/2-中/3-高 |
| `PlayIndex` | int | 正在播放的音频索引 (0=空闲/停止) |
| `DeviceStatus` | int | 设备状态 0-空闲 1-播放中 2-错误 |
| `ServerAddress` | int | Modbus 从站地址 |
| `ComPort` | QString | 串口号 |
| `BaudRate` | int | 波特率 |

**内部轮询循环**:

```
Read_HoldingRegisters(0, 124)
  → Read_HoldingRegisters_Done:
      解析 HR2 → g_Voice.DeviceStatus (0=空闲/1=播放中/2=错误)
      更新 UI 状态:
        Voice_Connected 指示灯
        Voice_Device_Status 文本 (空闲/播放中/错误)
        Voice_Current_Audio 显示 (PA001 格式)
        Voice_Volume_Display 显示 (低/中/高)
      → Write_HoldingRegisters_Action:
          检查 m_Set_HoldingRegisters[0] (音频索引)
          检查 m_Set_HoldingRegisters[1] (音量等级)
      → 循环 Read_HoldingRegisters(0, 124)
```

### 11.7 触发时序示例

```
工作流触发:
  Robot_App 进入采集状态 (workState=2)
    → m_Voice_Remote->Play_PA001()         // "正在巡检中，请注意安全"

    → m_Set_HoldingRegisters[0] = true

    → m_Voice_Remote->Play_PA006()         // "机械臂正在运行中，请注意"
```

## 十二、远程 IO 站控制协议 (XB6S-MT2002)

> XB6S-MT2002 总线耦合器，将远程 IO 模块通过 Modbus TCP 接入以太网。
> Robot_App 通过 `c_IO_Remote` 类实现 DI 采集、DO 输出、AO 控制。

### 12.1 硬件组成

| 模块型号 | 通道数 | Modbus 类型 | 用途 |
|---------|--------|-----------|------|
| XB6S-MT2002 | — | 总线耦合器 | 以太网主站 |
| XB6S-3200 | 16 路 DI | Discrete Inputs | 传感器/限位开关采集 |
| XB6S-3200B | 16 路 DO | Coils | 继电器/执行器控制 |
| XB6S-A04 | 4 路 AO | Holding Registers | 比例阀/变频器控制 |

### 12.2 Modbus 地址映射

| 数据类型 | 起始地址 | 数量 | 对应模块 |
|---------|---------|------|---------|
| Coils (DO) | 0 | 16 | XB6S-3200B |
| Discrete Inputs (DI) | 0 | 16 | XB6S-3200 |
| Input Registers (AI) | 0 | 0 | (未配备) |
| Holding Registers (AO) | 0 | 4 | XB6S-A04 |

### 12.3 数字量输出接线 (XB6S-3200B → 继电器)

```
Y01 (Coil 0) ──→ KA2 ──→ 三色报警灯 红色 (左前/左后/右前/右后 四组)
Y02 (Coil 1) ──→ KA3 ──→ 三色报警灯 黄色
Y03 (Coil 2) ──→ KA4 ──→ 三色报警灯 绿色
Y04 (Coil 3) ──→ KA5 ──→ 散热风扇
```

### 12.4 通信流程

```
Robot_App IO_Widget 初始化:
  → IO_Init() 创建 c_IO_Remote 线程
  → Modbus TCP 连接 (IP:192.168.1.10, Port:502)
  → 连接成功 → 启动轮询循环

轮询周期 (Coils → HoldingRegisters → DiscreteInputs → InputRegisters):
  1. Read_Coils(0, 16)
     → g_IO.Coils[0..15]  ←  线圈实际状态
     → Write_Coils_Action(): 检查是否有待处理写请求
     → Write_HoldingRegisters_Action(): 同上
  2. Read_HoldingRegisters(0, 4)  → g_IO.read_HoldingRegisters[0..3]
  3. Read_DiscreteInputs(0, 16)   → g_IO.DiscreteInputs[0..15] → 更新 UI 表
  4. Read_InputRegisters(0, 0)    → (空, 无模拟量输入模块)
  5. 循环回 1

写操作 (延迟写入, 读写验证):
  用户触发 Set_Alarm_Red()
    → m_Set_Coils[0] = true
    → 下一轮写入周期: emit Write_Coils(0, 1, {1})
    → 回读验证: Read_Coils_Done → 检查 g_IO.Coils[0]==1 → 确认完成
```
