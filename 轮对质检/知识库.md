## 一、系统架构总览

```
┌──────────────────────────────────────────────────────────────┐
│                     Robot_App (Qt C++ GUI)                   │
│                       主窗口: c_Robot_App_Widget              │
├──────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌───────────────┐  ┌──────────────┐ │
│  │Magic_Remote │  │ CGXi_Remote │  │ Lczh_Remote   │  │Robot_Client  │ │
│  │  (HTTP线程) │  │(Modbus线程) │  │(相机控制线程)  │  │  /Server     │ │
│  └──────┬──────┘  └──────┬──────┘  └───────┬───────┘  │(TCP/JSON线程)│ │
│         │                │                  │          └──────┬───────┘ │
│  ┌──────┴──────┐  ┌──────┴──────┐  ┌───────┴───────┐         │         │
│  │ HTTP_Client │  │ModeBusTCP   │  │ Scan_Remote   │    ┌────┴────┐    │
│  │(网络管理器) │  │  _Client    │  │ Scan_Monitor  │    │TCP_Clnt │    │
│  └─────────────┘  └─────────────┘  │ Scan_Tran     │    └─────────┘    │
│                                     │ (3线程)       │                  │
└─────────────────────────────────────┴───────────────┴──────────────────┘
│                 │                     │                    │
     HTTP REST         Modbus TCP           UDP x2 / TCP         TCP/JSON
     192.168.1.102:8888  从Communicate_DB读取  从Communicate_DB读取
          │                 │                     │                    │
     ┌────┴────┐      ┌────┴────┐          ┌─────┴─────┐        ┌────┴────┐
     │ 煜禾森   │      │  CGXi   │          │ 立创相机   │        │ 精扫/   │
     │ Magic    │      │ 机械臂  │          │  T1       │        │ 外部设备 │
     │ AMR底盘  │      │ 控制器  │          │ (结构光)   │        │         │
     └─────────┘      └─────────┘          └───────────┘        └─────────┘
```

**线程模型**：四层嵌套线程架构，每个设备通信模块运行在独立 `QThread` 中：
- `m_Magic_Remote_Thread` → `c_Magic_Remote` (HTTP 轮询，200ms 定时器)
  - └── `m_HTTP_Thread` → `c_HTTP_Client` (`QNetworkAccessManager`)
- `m_CGXi_Remote_Thread` → `c_CGXi_Remote` (Modbus 状态机，50ms 读周期)
  - └── `m_ModeBusTCP_Thread` → `c_ModeBusTCP_Client` (`QModbusTcpClient`)
- `m_Lczh_Remote_Thread` → `c_Lczh_Remote` (相机控制)
  - └── `m_Remote_Thread` → `m_Scan_Remote` (UDP 发送至相机 8001)
  - └── `m_Monitor_Thread` → `m_Scan_Monitor` (UDP 接收相机反馈 8000)
  - └── `m_Tran_Thread` → `m_Scan_Tran` (TCP Server 监听 9000，接收 CGXi 消息)
- 主线程：`System_Scan()` 10ms 递归 `QTimer::singleShot` 驱动 UI 刷新

**全局状态管理**：`c_Variable` 单例持有所有全局数据库：
- `g_Magic` (`s_Magic_DB`) — AMR 底盘状态
- `g_CGXi` (`s_CGXi_DB`) — 机械臂状态（含 1000 寄存器联合体映射）
- `g_Work_DB` (`s_Work_DB`) — 轮对质检任务状态与采集参数
- `g_Prec_Scan_120` (`s_Scan_DB`) — 立创相机通信状态（Remote/Monitor/Tran 三路连接）
- `g_Communicate_DB` (`QJsonObject`) — 系统通讯参数
- `g_Worry_List` / `g_Work_List` — 报警/巡检日志

## 二、机械臂 Modbus 通讯协议（CGXi）

### 2.1 通信方式

| 项目 | 值 |
|------|-----|
| 协议 | Modbus TCP（**SDK 未启用**，`robotapi.h` 已 include 但无 `cr_*` 调用） |
| 端口 | 从 `Communicate_DB.json` 的 `CGXi_Port` 读取（当前配置为 2323，**疑似 SDK 端口**，标准 Modbus 应为 502，需现场确认） |
| IP | 从 `Communicate_DB.json` 的 `CGXi_IP` 读取（当前配置 `192.168.6.6`） |
| 底层库 | Qt `QModbusTcpClient`（封装于 `ModeBusTCP_Client`） |
| 数据模型 | `s_CGXi_Date` 结构体（1000 个保持寄存器完整映射） |
| 连接策略 | `Init()` 后 `singleShot(1500ms)` 自动连接，断线后 `singleShot(3000ms)` 重连 |

### 2.2 寄存器分区映射

| 寄存器范围 | 内容 | 读写 | `s_CGXi_Date` 字段前缀 |
|-----------|------|------|----------------------|
| 0-99 | 基础状态与 IO | R/W | `version`, `standard_digital_*`, `configurable_digital_*`, `analog_*`, `tool_*` |
| 30-55 | 编码器与 RTC | R | `encoder_*`, `rtc_*` |
| 60-89 | 模式/错误/程序控制 | R/W | `robot_mode`, `program_status`, `clear_robot_error`, `target_program_*` |
| 100-199 | 网络参数 | R | `controller_ip`, `controller_mask` |
| 200-371 | 关节数据 (6轴×12组) | R | `joint_mode`, `joint_temperature`, `joint_actual/target_position/speed/acceleration/torque` |
| 500-611 | TCP 数据 (6维×10组) | R | `tcp_actual/target_position/speed/force` |
| 700-731 | TCP/负载/速度配置 | R/W | `tcp_flange_offset`, `payload_mass_and_cog`, `speed_ratio` |
| 800-821 | 点动运动控制 | W | `jog_type`, `jog_speed_ratio`, `jog_coordinate_system`, `jog_target_position` |
| 850-863 | 运动统计 | R | `total_motion_time`, `total_power`, `mechanical_power` 等 |

### 2.3 关键操作流程

**机器人使能（线圈）**：
```
Coil 1000=ON → Coil 1001=ON → Coil 1002=ON  → Coil 1003=ON
（清除错误）   （清除警告）    （上电）         （使能）
```
对应代码: `CGXi_Remote::Set_Coils_1000()` ~ `Set_Coils_1003()`

**程序控制（保持寄存器）**：
```
寄存器 88: 设置目标程序索引
寄存器 89: 1=启动, 2=暂停, 3=停止
```
对应代码: `Set_HoldingRegisters_88()`, `Set_HoldingRegisters_89()`

**点动控制**：
```
寄存器 800: 点动类型 → 801-802: 速度比例 → 803: 坐标系 → 810-821: 目标位姿(6×float=12 reg)
```
对应代码: `Set_HoldingRegisters_800()` ~ `Set_HoldingRegisters_810()`

### 2.4 数据类型转换

| 原始类型 | 寄存器数量 | 编码方式 | 转换函数 |
|---------|-----------|---------|---------|
| `BOOL` | 1 reg (每个bit对应1路) | 位映射 | 直接位操作 |
| `quint16` | 1 reg | 大端 | 直接读取 |
| `qint16` | 1 reg | 大端补码 | 直接读取 |
| `float` | 2 reg | 小端IEEE754 (低16位在前) | `BitToFloat()` |
| `qint64` / `quint32` | 2-4 reg | 大端无符号 | `BitToInt()` |

## 四、数据流与生命周期

### 4.1 系统初始化流程

```
main() → c_Robot_App_Widget 构造
  → Variable_Init()    // 初始化单例变量
  → Variable_DB()      // 绑定UI控件到全局变量
  → QTimer::singleShot 链:
    100ms → System_Scan() 首次调用（10ms递归 self-singleShot）
    100ms → Magic_Init()  // 创建 HTTP_Client → Magic_Remote → 启动HTTP线程
    200ms → CGXi_Init()   // 创建 ModeBusTCP_Client → CGXi_Remote → 启动Modbus线程
    400ms → Lczh_Init()   // 创建 c_Lczh_Remote → 启动相机控制线程
    Magic_Init 内联调用 Magic_DB() + Magic_Button()
    CGXi_Init 内联调用 CGXi_DB() + CGXi_Button()
  → System_Scan() 每帧执行 Magic_Scan() + CGXi_Scan()（无独立 Controller_Scan）
```

### 4.2 Magic 连接与数据流

```
用户触发 Connect → Magic_Remote::Connect_Device(ip, port)
  → HTTP_Client::Connect_Device(ip, port, "/yhs-robot/")
  → Connect_Done(url)
  → Login(userCode, password)  →  POST /auth/token
  → Set_Token_Done(token)
  → 启动 m_PollTimer (200ms) → Poll_Status() 轮询
  → 每次 Response 回调 On_Response_Done → 更新 g_Magic.ReadData
  → emit Update_Magic_DB() → UI 刷新
```

### 4.3 CGXi Modbus 读写流

```
CGXi_Remote::Connect() → emit Connect_Device(ip, port)
  → ModeBusTCP_Client::Connect_Device() → QModbusTcpClient 连接
  → 成功后启动 50ms 定时循环:
    Read_HoldingRegisters_Action() → 分批次读取 0-999 寄存器
    → Read_HoldingRegisters_Done(data) → 解析至 g_CGXi.read_HoldingRegisters[]
    → 结构体 s_CGXi_Date 通过 union 自动映射数据

写操作（按需触发）:
  Write_Coils_Action()      → 线圈写入 1000-1007
  Write_HoldingRegisters_Action() → 寄存器标记区写入
```

## 五、关键设计模式与注意事项

### 5.1 CGXi Modbus 寄存器映射技巧

`s_CGXi_DB` 使用匿名 `union` 实现数组与结构体的零拷贝映射:
```cpp
union {
    quint16 read_HoldingRegisters[1000];  // 原始寄存器数组
    s_CGXi_Date data;                      // 结构化访问
};
```
这样读取寄存器后，可直接通过 `data.joint_actual_position[0]` 访问关节1实际位置，无需手动解析。

### 5.2 CGXi Modbus 寄存器布局关键规则

- **float 类型**: 占用2个寄存器（32位），排列为 **低16位在前、高16位在后**，与小端CPU的float内存布局天然兼容
- **int32/uint32 类型**: 占用2个寄存器，**高16位在前、低16位在后**（大端）
- **int64 类型**: 占用4个寄存器（如编码器计数值）

### 5.3 Magic HTTP Token 机制

- 首次调用 `Login()` 获取 token
- 后续所有请求 header 中携带 `Authorization: Bearer {token}`
- Token 存储于 `g_Magic.token`
- 如 401 未授权，需重新 Login

### 5.4 线程安全

- 每个通讯模块运行在独立 QThread 中
- 跨线程通信使用 Qt 信号槽（`Qt::QueuedConnection` 自动排队）
- 全局变量 `c_Variable` 使用 `QReadWriteLock` 保护（`g_lock`）：工作线程写 (`On_Response_Done` 中 `QWriteLocker`)，主线程读 (`Magic_Scan` 中 `QReadLocker`)
- **注意**：Qt 5.12.1 无 `QReadWriteLocker` 类，单例工厂使用手动 `lockForWrite()`/`unlock()` 代替

### 5.5 错误处理模式

```
设备层错误 → Status 信号 → Write_Worry_List() / Write_Magic_List() / Write_CGXi_List()
  → g_Worry_List (QJsonArray) → UI 报警列表显示
```

## 十一、立创相机采集通信模块（Lczh_Remote）

### 11.1 整体结构

`c_Lczh_Remote : c_Object` 管理立创结构光相机 T1 的图像采集流程，位于 4 层协议的第 2 层（上位机 ↔ 相机）。

```
┌─────────────────────────────────────────────────────────────┐
│                  c_Lczh_Remote (相机控制线程)                 │
│  ┌──────────────┐  ┌──────────────┐  ┌────────────────────┐ │
│  │ m_Scan_Remote │  │ m_Scan_Monit│  │ m_Scan_Tran       │ │
│  │ UDP 发送      │  │ UDP 接收     │  │ TCP Server :9000  │ │
│  │  → 相机:8001  │  │ ← 相机:8000  │  │ ← CGXi 连接      │ │
│  └──────┬───────┘  └──────┬───────┘  └────────┬───────────┘ │
│         │                  │                     │            │
│    Signal_Remote_Write  Read_String_Done    Read_Done        │
└─────────┼──────────────────┼─────────────────────┼────────────┘
          │                  │                     │
     ┌────┴────┐        ┌────┴────┐          ┌────┴────┐
     │ 相机 T1  │        │ 相机 T1  │          │ CGXi    │
     │ UDP:8001 │        │ UDP:8000 │          │ TCP Clnt│
     └─────────┘        └─────────┘          └─────────┘
```

**线程模型**：4 个线程的 3 层嵌套：
- `m_Lczh_Remote_Thread` — Lczh_Remote 控制逻辑（协议解析、指令合成、状态管理）
  - `m_Remote_Thread` → `m_Scan_Remote` (UDP 发送)
  - `m_Monitor_Thread` → `m_Scan_Monitor` (UDP 接收)
  - `m_Tran_Thread` → `m_Scan_Tran` (TCP Server)

### 11.2 通信参数

| 通道 | 协议 | 默认端口 | 存储键（Communicate_DB.json） | 说明 |
|------|------|---------|------------------------------|------|
| Remote (发送) | UDP | 8001 | `Camera_IP`, `Camera_Port` | Robot_App → 相机控制指令 |
| Monitor (接收) | UDP | 8000 | `Monitor_Port` | 相机 → Robot_App 反馈 |
| Tran (监听) | TCP Server | 9000 | `Tran_Port` | CGXi → Robot_App 采集指令 |

### 11.3 信号桥接（跨线程通信）

所有跨线程调用通过 `QObject::connect` 以信号-槽方式转发，`Lczh_Remote`（线程 A）→ 三个 Worker（线程 B/C/D）自动 `Qt::QueuedConnection`：

| 信号 | 连接到的 Worker Slot | 方向 |
|------|---------------------|------|
| `Signal_Remote_Connect(ip, port)` | `m_Scan_Remote::Connect_Device` | 绑定 UDP 发送端 |
| `Signal_Remote_Write(ip, port, value)` | `m_Scan_Remote::Write_String` | 发送指令到相机 |
| `Signal_Remote_Disconnect()` | `m_Scan_Remote::Disconnect_Device` | 断开 UDP |
| `Signal_Monitor_Connect(ip, port)` | `m_Scan_Monitor::Connect_Device` | 绑定 UDP 接收端 |
| `Signal_Monitor_Disconnect()` | `m_Scan_Monitor::Disconnect_Device` | 断开 UDP |
| `Signal_Tran_Connect(ip, port)` | `m_Scan_Tran::Connect_Device` | 启动 TCP Server |
| `Signal_Tran_Disconnect()` | `m_Scan_Tran::Disconnect_Device` | 停止 TCP Server |
| `Signal_Tran_Write(value)` | `m_Scan_Tran::Write_String` | 应答 CGXi |

Worker → Lczh_Remote 方向（回来信号）：
| Worker 信号 | 连接到 Lczh_Remote Slot | 说明 |
|------------|------------------------|------|
| `m_Scan_Tran::Read_Done` | → `On_CGXi_Data` (粘包处理) | CGXi 原始数据 |
| `m_Scan_Monitor::Read_String_Done` | → `On_Camera_Feedback` | 相机反馈 |
| `m_Scan_*::Connect_Done` | → `Scan_*_Connect_Done` | 三路握手状态 |
| `m_Scan_*::Disconnect_Done` | → `Scan_*_Disconnect_Done` | 断开通知 |
| `m_Scan_Tran::Listen_Done` | → `Scan_Tran_Listen_Done` | TCP 监听成功 |
| `m_Scan_*::Status` | → 状态日志（含 TCP_Status 转换） | 调试日志 |

### 11.4 三路握手（Ready 机制）

三个通道全部连接就绪后发射 `Set_Working()`，任一个断开发射 `Set_Default()`：

```
Scan_Remote_Connect_Done → Remote_Connected = true
Scan_Monitor_Connect_Done → Monitor_Connected = true
Scan_Tran_Connect_Done → Tran_Connected = true, num++
                                    ↓
               全部 true? → emit Set_Working()

任一 Scan_*_Disconnect_Done → 对应 _Connected = false → emit Set_Default()
```

状态位存储在 `g_Prec_Scan_120`：
- `Remote_Connected` — UDP 发送通道就绪
- `Monitor_Connected` — UDP 接收通道就绪
- `Tran_Connected` — TCP 客户端已连接
- `Tran_Listen` — TCP 服务器监听中
- `num` — 当前 TCP 客户端数量

### 11.5 TCP 粘包处理

CGXi 通过 Socket TCP 发送指令，可能多指令合并或半包到达。

```
On_CGXi_Data(QByteArray data)
  → m_CGXi_Buffer.append(data)
  → while (找到 '!') {
      提取 msg 从缓冲区开头到 '!'（含）
      从缓冲区移除已提取部分
      On_CGXi_Message(msg)  // 逐条处理完整消息
    }
  → 不完整数据留在缓冲区
```

定界符 `!` 是 CGXi 协议所有消息的固定结尾字符。

### 11.6 消息校验

On_CGXi_Message 和 On_Camera_Feedback 对每条消息做 13 项校验，任一不通过则 `emit Status()` 上报 UI 并 return：

| 消息类型 | 校验项 |
|---------|--------|
| 所有 | 空字节、UTF-8 解码失败 |
| START | `sendUnit`/`startTime`/`wheelsetNo`/`axleNo`/`taskId` 非空 |
| CHANG&GAIN | 以 `!` 结尾、含 `&PHOTO&`、字段数≥5、GAIN 范围 0-100、程序索引≥0、部件名称非空 |
| PHOTO&FINISH | 精确匹配（无需额外校验） |
| 相机 COMPLETE | 精确匹配或 `startsWith`（兼容带后缀） |
| 相机 ERROR | 精确匹配或 `startsWith` |
| 未知消息 | 兜底 `emit Status("未知指令")` |

### 11.7 手动触发槽函数

| 槽函数 | 发送指令 | 附加动作 |
|--------|---------|---------|
| `Start_Scan()` | `START&{sendUnit}&{date}&{time}&{wsNo}{axNo}&{taskId}` | `workState=2`, `m_LastGain=-1` |
| `Collector()` | `PHOTO&{wsNo}{axNo}&{完整文件名}` | 从 `g_Work_DB` 读取参数 |
| `Finish_Scan()` | `PHOTO&FINISH` | `workState=4` |
| `Change_GAIN(gain)` | `CHANGE&GAIN&{gain}` | 校验 0-100，更新 `m_LastGain` |
