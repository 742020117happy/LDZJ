**线程模型**：六模块独立线程架构，每个设备通信模块运行在独立 `QThread` 中：
- `m_Magic_Remote_Thread` → `c_Magic_Remote` (HTTP 轮询，200ms 定时器)
  - └── `m_HTTP_Thread` → `c_HTTP_Client` (`QNetworkAccessManager`)
- `m_CGXi_Remote_Thread` → `c_CGXi_Remote` (Modbus 状态机，50ms 读周期)
  - └── `m_ModeBusTCP_Thread` → `c_ModeBusTCP_Client` (`QModbusTcpClient`)
- `m_Lczh_Remote_Thread` → `c_Lczh_Remote` (相机控制)
  - └── `m_Remote_Thread` → `m_Scan_Remote` (UDP 发送至相机 8001)
  - └── `m_Monitor_Thread` → `m_Scan_Monitor` (UDP 接收相机反馈 8000)
  - └── `m_Tran_Thread` → `m_Scan_Tran` (TCP Server 监听 9000，接收 CGXi 消息)
- `m_Voice_Remote_Thread` → `c_Voice_Remote` (语音播报，Modbus RTU 串口)
  - └── `m_ModeBusRTU_Thread` → `c_ModeBusRTU_Master` (`QSerialPort`)
- `m_IO_Remote_Thread` → `c_IO_Remote` (远程 IO 站，Modbus TCP 轮询)
  - └── `c_ModeBusTCP_Client` (与 CGXi 共用 Modbus TCP 底层)
- 主线程：`System_Scan()` 10ms 递归 `QTimer::singleShot` 驱动 UI 刷新

**全局状态管理**：`c_Variable` 单例持有所有全局数据库：
- `g_Magic` (`s_Magic_DB`) — AMR 底盘状态
- `g_CGXi` (`s_CGXi_DB`) — 机械臂状态（含 1000 寄存器联合体映射）
- `g_Work_DB` (`s_Work_DB`) — 轮对质检任务状态与采集参数
- `g_Prec_Scan_120` (`s_Scan_DB`) — 立创相机通信状态（Remote/Monitor/Tran 三路连接）
- `g_Communicate_DB` (`QJsonObject`) — 系统通讯参数
- `g_Worry_List` / `g_Work_List` — 报警/巡检日志
- `g_Voice` (`s_Voice_DB`) — 语音播报器状态（连接/音量/播放索引）
- `g_IO` (`s_IO_DB`) — 远程 IO 站状态（Coils / DiscreteInputs / InputRegisters / HoldingRegisters 各 1000 字缓存）

## 二、机械臂 Modbus 通讯协议（CGXi）

### 2.1 通信方式

| 项目 | 值 |
|------|-----|
| 协议 | Modbus TCP（**SDK 未启用**，`robotapi.h` 已 include 但无 `cr_*` 调用） |
| 端口 | 从 `Communicate_DB.json` 的 `CGXi_Port` 读取（当前配置为 2323，**疑似 SDK 端口**，标准 Modbus 应为 502，需现场确认） |
| IP | 从 `Communicate_DB.json` 的 `CGXi_IP` 读取（当前配置 `192.168.6.6`） |
| 底层库 | Qt `QModbusTcpClient`（封装于 `ModeBusTCP_Client`） |
| 数据模型 | `s_CGXi_Date` 结构体（1000 个保持寄存器完整映射） |
| 连接策略 | `Init()` 后 `singleShot(1500ms)` 自动连接，断线后 `singleShot(3000ms)` 重连 |

### 2.2 寄存器分区映射

| 寄存器范围 | 内容 | 读写 | `s_CGXi_Date` 字段前缀 |
|-----------|------|------|----------------------|
| 0-99 | 基础状态与 IO | R/W | `version`, `standard_digital_*`, `configurable_digital_*`, `analog_*`, `tool_*` |
| 30-55 | 编码器与 RTC | R | `encoder_*`, `rtc_*` |
| 60-89 | 模式/错误/程序控制 | R/W | `robot_mode`, `program_status`, `clear_robot_error`, `target_program_*` |
| 100-199 | 网络参数 | R | `controller_ip`, `controller_mask` |
| 200-371 | 关节数据 (6轴×12组) | R | `joint_mode`, `joint_temperature`, `joint_actual/target_position/speed/acceleration/torque` |
| 500-611 | TCP 数据 (6维×10组) | R | `tcp_actual/target_position/speed/force` |
| 700-731 | TCP/负载/速度配置 | R/W | `tcp_flange_offset`, `payload_mass_and_cog`, `speed_ratio` |
| 800-821 | 点动运动控制 | W | `jog_type`, `jog_speed_ratio`, `jog_coordinate_system`, `jog_target_position` |
| 850-863 | 运动统计 | R | `total_motion_time`, `total_power`, `mechanical_power` 等 |

### 2.3 关键操作流程

**机器人使能（线圈）**：
```
Coil 1000=ON → Coil 1001=ON → Coil 1002=ON  → Coil 1003=ON
（清除错误）   （清除警告）    （上电）         （使能）
```
对应代码: `CGXi_Remote::Set_Coils_1000()` ~ `Set_Coils_1003()`

**程序控制（保持寄存器）**：
```
寄存器 88: 设置目标程序索引
寄存器 89: 1=启动, 2=暂停, 3=停止
```
对应代码: `Set_HoldingRegisters_88()`, `Set_HoldingRegisters_89()`

**点动控制**：
```
寄存器 800: 点动类型 → 801-802: 速度比例 → 803: 坐标系 → 810-821: 目标位姿(6×float=12 reg)
```
对应代码: `Set_HoldingRegisters_800()` ~ `Set_HoldingRegisters_810()`

### 2.4 数据类型转换

| 原始类型 | 寄存器数量 | 编码方式 | 转换函数 |
|---------|-----------|---------|---------|
| `BOOL` | 1 reg (每个bit对应1路) | 位映射 | 直接位操作 |
| `quint16` | 1 reg | 大端 | 直接读取 |
| `qint16` | 1 reg | 大端补码 | 直接读取 |
| `float` | 2 reg | 小端IEEE754 (低16位在前) | `BitToFloat()` |
| `qint64` / `quint32` | 2-4 reg | 大端无符号 | `BitToInt()` |

## 四、数据流与生命周期

### 4.1 系统初始化流程

```
main() → c_Robot_App_Widget 构造
  → Variable_Init()    // 初始化单例变量
  → Variable_DB()      // 绑定UI控件到全局变量
  → QTimer::singleShot 链:
    100ms → System_Scan() 首次调用（10ms递归 self-singleShot）
    100ms → Magic_Init()  // 创建 HTTP_Client → Magic_Remote → 启动HTTP线程
    200ms → CGXi_Init()   // 创建 ModeBusTCP_Client → CGXi_Remote → 启动Modbus线程
    400ms → Lczh_Init()   // 创建 c_Lczh_Remote → 启动相机控制线程
    Magic_Init 内联调用 Magic_DB() + Magic_Button()
    CGXi_Init 内联调用 CGXi_DB() + CGXi_Button()
  → System_Scan() 每帧执行 Magic_Scan() + CGXi_Scan()（无独立 Controller_Scan）
```

### 4.2 Magic 连接与数据流

```
用户触发 Connect → Magic_Remote::Connect_Device(ip, port)
  → HTTP_Client::Connect_Device(ip, port, "/yhs-robot/")
  → Connect_Done(url)
  → Login(userCode, password)  →  POST /auth/token
  → Set_Token_Done(token)
  → 启动 m_PollTimer (200ms) → Poll_Status() 轮询
  → 每次 Response 回调 On_Response_Done → 更新 g_Magic.ReadData
  → emit Update_Magic_DB() → UI 刷新
```

### 4.3 CGXi Modbus 读写流

```
CGXi_Remote::Connect() → emit Connect_Device(ip, port)
  → ModeBusTCP_Client::Connect_Device() → QModbusTcpClient 连接
  → 成功后启动 50ms 定时循环:
    Read_HoldingRegisters_Action() → 分批次读取 0-999 寄存器
    → Read_HoldingRegisters_Done(data) → 解析至 g_CGXi.read_HoldingRegisters[]
    → 结构体 s_CGXi_Date 通过 union 自动映射数据

写操作（按需触发）:
  Write_Coils_Action()      → 线圈写入 1000-1007
  Write_HoldingRegisters_Action() → 寄存器标记区写入
```

## 五、关键设计模式与注意事项

### 5.1 CGXi Modbus 寄存器映射技巧

`s_CGXi_DB` 使用匿名 `union` 实现数组与结构体的零拷贝映射:
```cpp
union {
    quint16 read_HoldingRegisters[1000];  // 原始寄存器数组
    s_CGXi_Date data;                      // 结构化访问
};
```
这样读取寄存器后，可直接通过 `data.joint_actual_position[0]` 访问关节1实际位置，无需手动解析。

### 5.2 CGXi Modbus 寄存器布局关键规则

- **float 类型**: 占用2个寄存器（32位），排列为 **低16位在前、高16位在后**，与小端CPU的float内存布局天然兼容
- **int32/uint32 类型**: 占用2个寄存器，**高16位在前、低16位在后**（大端）
- **int64 类型**: 占用4个寄存器（如编码器计数值）

### 5.3 Magic HTTP Token 机制

- 首次调用 `Login()` 获取 token
- 后续所有请求 header 中携带 `Authorization: Bearer {token}`
- Token 存储于 `g_Magic.token`
- 如 401 未授权，需重新 Login

### 5.4 线程安全

- 每个通讯模块运行在独立 QThread 中
- 跨线程通信使用 Qt 信号槽（`Qt::QueuedConnection` 自动排队）
- 全局变量 `c_Variable` 使用 `QReadWriteLock` 保护（`g_lock`）：工作线程写 (`On_Response_Done` 中 `QWriteLocker`)，主线程读 (`Magic_Scan` 中 `QReadLocker`)
- **注意**：Qt 5.12.1 无 `QReadWriteLocker` 类，单例工厂使用手动 `lockForWrite()`/`unlock()` 代替

### 5.5 错误处理模式

```
设备层错误 → Status 信号 → Write_Worry_List() / Write_Magic_List() / Write_CGXi_List()
  → g_Worry_List (QJsonArray) → UI 报警列表显示
```

### 11.2 IO 触发模式（触点/开关量）

所有 IO 型号（AW-IO / GF-IO / GFY-IO / GJB-1838）均支持 8 路 IO 触点 K1-K8：
- 每路触点可独立触发播放指定的音频文件
- 每路最多绑定 255 个音频文件（PA001-PA255）
- 短接触点 K1 即播放对应编号的音频

### 11.3 RS485 / RS232 串口协议

**通讯参数**：9600 bps（默认），8 数据位，地址范围 0x01-0xFF

**指令格式**：
```
FF  Cmd  Data...  CS  AF
│    │      │      │   │
帧头  命令   参数   校验  帧尾
```
- 帧头：`0xFF`（固定）
- 帧尾：`0xAF`（固定）
- 校验：第2字节到倒数第2字节的异或和

**命令列表（5W/15W 通用）**：

| 命令 | 方向 | 功能 | 应答 |
|------|------|------|------|
| `0x0E` | 查询 | 查询设备状态 | `0x0D` + 状态 |
| `0x1E` | 设置 | 设置波特率 | `0x1E` 回声 + `0x1D` 成功/ `0x1F` 失败 |
| `0x2E` | 设置 | 设置音量 (1-3级) | `0x2E` 回声 + `0x2D` 成功/ `0x2F` 失败 |
| `0x3E` | 设置 | 设置循环模式 | `0x3E` 回声 + `0x3D`/`0x3F` |
| `0x4E` | 设置 | 设置设备地址 | `0x4E` 回声 + `0x4D`/`0x4F` |
| `0x5E` | 触发 | 播放指定音频 | `0x5E` 回声 + `0x5D`/`0x5F` |

**播放示例（播放序号 2 的音频）**：
```
FF 5E 00 00 02 A3 AF
```

**波特率设置**：
| 值 | 波特率 | 示例指令 |
|----|--------|---------|
| 0x01 | 4800 | `FF 1E 00 00 01 D0 AF` |
| 0x02 | 9600 (默认) | `FF 1E 00 00 02 D3 AF` |
| 0x08 | 115200 |  |

### 11.4 RS485 / RS232 Modbus 协议

**寄存器地址映射**：

| 寄存器地址 | 名称 | 功能 |
|-----------|------|------|
| `0x0010` | 波特率设置 | 读/写波特率值 |
| `0x0011` | 音量设置 | 音量 1-3 |
| `0x0012` | 循环模式 | 1:单曲 2:循环 3:随机 |
| `0x0013` | 播放次数 | 指定播放次数 |
| `0x0014` | 保留 | — |
| `0x0003` | 播放编号 | 写入音频编号触发播放 |
| `0x0004` | 循环次数 | 指定重复播放次数 |
| `0x0005` | 播放状态 | 读取:0=停止 1=播放 2=暂停 |

**功能码**：`0x03`(读), `0x06`(写单), `0x10`(写多)

### 11.5 CAN-bus 协议

**通信参数**：CAN ID 发送 `0x300` / 接收 `0x301`，速率 500 Kbps，DLC 8

**指令格式**：帧头 `0xAE`，帧尾 `0xAF`，校验同串口

**命令列表**：`0x0E`(查询), `0x1E`(CAN波特率), `0x2E`(使能), `0x3E`(接收ID), `0x4E`(发送ID), `0x5E`(播放)

**CAN-bus 音频触发**：`0x51`(播放), `0x52`(停止), `0x53`(循环次数), `0x54`(音量)

### 11.6 433MHz / WiFi 无线协议

**发送**：`FC ADDR CMD DATA RESV×8 CS`（13字节）
**接收**：`FD ADDR CMD STATUS RESV×8 CS`
- `0xAA` 播放 / `0xAB` 停止
- 状态字节反馈 NPN_IO、OUT、IO 引脚电平

### 11.7 音频文件管理

- 通过 MicroUSB 以 U 盘模式复制 WAV/MP3
- 文件名格式 `PA001`~`PA255`
- 最多 255 段，128MB/512MB Flash

### 11.8 与 Robot_App 集成思路

| 方式 | 硬件 | 接入路径 | 复杂度 |
|------|------|---------|--------|
| RS485 Modbus | PPY-AW-485 | `QSerialPort` Modbus RTU | 中 |
| TCP/WiFi | PPY-GJB-WIFI | 局域网自定义协议 | 低 |
| IO 触发 | PPY-AW-IO | PLC/继电器触发，无需改软件 | 无 |
| CAN-bus | PPY-AW-CAN | CAN 转 USB | 高 |

## 十二、立创相机采集通信模块（Lczh_Remote）

### 11.1 整体结构

`c_Lczh_Remote : c_Object` 管理立创结构光相机 T1 的图像采集流程，位于 4 层协议的第 2 层（上位机 ↔ 相机）。

```
┌─────────────────────────────────────────────────────────────┐
│                  c_Lczh_Remote (相机控制线程)                 │
│  ┌──────────────┐  ┌──────────────┐  ┌────────────────────┐ │
│  │ m_Scan_Remote │  │ m_Scan_Monit│  │ m_Scan_Tran       │ │
│  │ UDP 发送      │  │ UDP 接收     │  │ TCP Server :9000  │ │
│  │  → 相机:8001  │  │ ← 相机:8000  │  │ ← CGXi 连接      │ │
│  └──────┬───────┘  └──────┬───────┘  └────────┬───────────┘ │
│         │                  │                     │            │
│    Signal_Remote_Write  Read_String_Done    Read_Done        │
└─────────┼──────────────────┼─────────────────────┼────────────┘
          │                  │                     │
     ┌────┴────┐        ┌────┴────┐          ┌────┴────┐
     │ 相机 T1  │        │ 相机 T1  │          │ CGXi    │
     │ UDP:8001 │        │ UDP:8000 │          │ TCP Clnt│
     └─────────┘        └─────────┘          └─────────┘
```

**线程模型**：4 个线程的 3 层嵌套：
- `m_Lczh_Remote_Thread` — Lczh_Remote 控制逻辑（协议解析、指令合成、状态管理）
  - `m_Remote_Thread` → `m_Scan_Remote` (UDP 发送)
  - `m_Monitor_Thread` → `m_Scan_Monitor` (UDP 接收)
  - `m_Tran_Thread` → `m_Scan_Tran` (TCP Server)

### 11.2 通信参数

| 通道 | 协议 | 默认端口 | 存储键（Communicate_DB.json） | 说明 |
|------|------|---------|------------------------------|------|
| Remote (发送) | UDP | 8001 | `Camera_IP`, `Camera_Port` | Robot_App → 相机控制指令 |
| Monitor (接收) | UDP | 8000 | `Monitor_Port` | 相机 → Robot_App 反馈 |
| Tran (监听) | TCP Server | 9000 | `Tran_Port` | CGXi → Robot_App 采集指令 |

### 11.3 信号桥接（跨线程通信）

所有跨线程调用通过 `QObject::connect` 以信号-槽方式转发，`Lczh_Remote`（线程 A）→ 三个 Worker（线程 B/C/D）自动 `Qt::QueuedConnection`：

| 信号 | 连接到的 Worker Slot | 方向 |
|------|---------------------|------|
| `Signal_Remote_Connect(ip, port)` | `m_Scan_Remote::Connect_Device` | 绑定 UDP 发送端 |
| `Signal_Remote_Write(ip, port, value)` | `m_Scan_Remote::Write_String` | 发送指令到相机 |
| `Signal_Remote_Disconnect()` | `m_Scan_Remote::Disconnect_Device` | 断开 UDP |
| `Signal_Monitor_Connect(ip, port)` | `m_Scan_Monitor::Connect_Device` | 绑定 UDP 接收端 |
| `Signal_Monitor_Disconnect()` | `m_Scan_Monitor::Disconnect_Device` | 断开 UDP |
| `Signal_Tran_Connect(ip, port)` | `m_Scan_Tran::Connect_Device` | 启动 TCP Server |
| `Signal_Tran_Disconnect()` | `m_Scan_Tran::Disconnect_Device` | 停止 TCP Server |
| `Signal_Tran_Write(value)` | `m_Scan_Tran::Write_String` | 应答 CGXi |

Worker → Lczh_Remote 方向（回来信号）：
| Worker 信号 | 连接到 Lczh_Remote Slot | 说明 |
|------------|------------------------|------|
| `m_Scan_Tran::Read_Done` | → `On_CGXi_Data` (粘包处理) | CGXi 原始数据 |
| `m_Scan_Monitor::Read_String_Done` | → `On_Camera_Feedback` | 相机反馈 |
| `m_Scan_*::Connect_Done` | → `Scan_*_Connect_Done` | 三路握手状态 |
| `m_Scan_*::Disconnect_Done` | → `Scan_*_Disconnect_Done` | 断开通知 |
| `m_Scan_Tran::Listen_Done` | → `Scan_Tran_Listen_Done` | TCP 监听成功 |
| `m_Scan_*::Status` | → 状态日志（含 TCP_Status 转换） | 调试日志 |

### 11.4 三路握手（Ready 机制）

三个通道全部连接就绪后发射 `Set_Working()`，任一个断开发射 `Set_Default()`：

```
Scan_Remote_Connect_Done → Remote_Connected = true
Scan_Monitor_Connect_Done → Monitor_Connected = true
Scan_Tran_Connect_Done → Tran_Connected = true, num++
                                    ↓
               全部 true? → emit Set_Working()

任一 Scan_*_Disconnect_Done → 对应 _Connected = false → emit Set_Default()
```

状态位存储在 `g_Prec_Scan_120`：
- `Remote_Connected` — UDP 发送通道就绪
- `Monitor_Connected` — UDP 接收通道就绪
- `Tran_Connected` — TCP 客户端已连接
- `Tran_Listen` — TCP 服务器监听中
- `num` — 当前 TCP 客户端数量

### 11.5 TCP 粘包处理

CGXi 通过 Socket TCP 发送指令，可能多指令合并或半包到达。

```
On_CGXi_Data(QByteArray data)
  → m_CGXi_Buffer.append(data)
  → while (找到 '!') {
      提取 msg 从缓冲区开头到 '!'（含）
      从缓冲区移除已提取部分
      On_CGXi_Message(msg)  // 逐条处理完整消息
    }
  → 不完整数据留在缓冲区
```

定界符 `!` 是 CGXi 协议所有消息的固定结尾字符。

### 11.6 消息校验

On_CGXi_Message 和 On_Camera_Feedback 对每条消息做 13 项校验，任一不通过则 `emit Status()` 上报 UI 并 return：

| 消息类型 | 校验项 |
|---------|--------|
| 所有 | 空字节、UTF-8 解码失败 |
| START | `sendUnit`/`startTime`/`wheelsetNo`/`axleNo`/`taskId` 非空 |
| CHANG&GAIN | 以 `!` 结尾、含 `&PHOTO&`、字段数≥5、GAIN 范围 0-100、程序索引≥0、部件名称非空 |
| PHOTO&FINISH | 精确匹配（无需额外校验） |
| 相机 COMPLETE | 精确匹配或 `startsWith`（兼容带后缀） |
| 相机 ERROR | 精确匹配或 `startsWith` |
| 未知消息 | 兜底 `emit Status("未知指令")` |

### 11.7 手动触发槽函数

| 槽函数 | 发送指令 | 附加动作 |
|--------|---------|---------|
| `Start_Scan()` | `START&{sendUnit}&{date}&{time}&{wsNo}{axNo}&{taskId}` | `workState=2`, `m_LastGain=-1` |
| `Collector()` | `PHOTO&{wsNo}{axNo}&{完整文件名}` | 从 `g_Work_DB` 读取参数 |
| `Finish_Scan()` | `PHOTO&FINISH` | `workState=4` |
| `Change_GAIN(gain)` | `CHANGE&GAIN&{gain}` | 校验 0-100，更新 `m_LastGain` |

### 11.8 全局数据结构


## 十六、语音报警器 RS485 控制方案

> 适用场景：**PPY-AW-485**（5W）或 **PPY-GF-485**/**PPY-GFY-485**（15W）型号，通过 RS485 总线连接至 MIC-770 的 COM 串口。

### 16.1 硬件接线

```
MIC-770 COM port (DB9 Male)
  Pin 3: TXD(A)  ───────────────── 语音播报器 485-A
  Pin 8: RXD(B)  ───────────────── 语音播报器 485-B
  Pin 5: GND     ───────────────── 语音播报器 GND
```

**MIC-770 COM1/COM2**（板载 RS-232/422/485 自适应）：
| 引脚 | RS-485 半双工信号 |
|------|------------------|
| 1 | D- |
| 2 | D+ |
| 5 | GND |

> 推荐使用 COM1，通过 BIOS 或软件切换为 RS-485 模式。
> 485 总线两端需加 120Ω 终端电阻（播报器端内部可选，MIC-770 端需外部焊接）。
> 线缆推荐 2 芯双绞屏蔽线，屏蔽层单端接地。
> 距离不超过 100m 时 9600 bps 可可靠通信。

### 16.2 协议选择

| 协议 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **自定义 FF/AF 协议** | 指令简洁，校验简单，与 Robot_App 现有风格一致 | 需手动组包/解包 | ⭐⭐⭐ 推荐 |
| Modbus RTU | 标准化，可共用现有 Modbus 代码 | 需要额外 CRC16 计算，开销大 | ⭐⭐ |

**推荐使用自定义 FF/AF 协议**，理由：
- 播报场景只有"播放音频"一个核心操作，无需复杂的寄存器读写
- 指令仅 6 字节，效率高
- 校验算法（异或和）简单，适合嵌入式

### 16.4 指令封装

**核心 API**（只需一个播放命令）：

```
播放音频 n：FF 5E 00 00 [n] [CS] AF
停止播放：  FF 5E 00 00 00  A0  AF  (n=0 表示停止)
设置音量：  FF 2E 00 00 [vol] [CS] AF   (vol=1/2/3)
查询状态：  FF 0E 00 00 00 [CS] AF
```

其中校验 CS = `0xFF XOR 0x5E XOR 0x00 XOR 0x00 XOR n`（从帧头后第1字节异或到数据末尾），对于播放命令简化为 `CS = 0xA1 XOR n`。

**Python 校验计算**：
```python
def voice_cmd(cmd, *data):
    buf = bytes([0xFF, cmd] + list(data) + [0xAF])
    cs = 0xFF
    for b in buf[1:-2]:  # 从 cmd 到 data 末尾
        cs ^= b
    buf[-2] = cs
    return buf
```

```cpp
// Voice_Remote.cpp
#include "Voice_Remote.h"

c_Voice_Remote::c_Voice_Remote(QObject *parent) : QObject(parent)
{
    m_Serial = new QSerialPort(this);
    connect(m_Serial, &QSerialPort::readyRead, this, &c_Voice_Remote::On_ReadyRead);
}

bool c_Voice_Remote::Open(const QString &portName, int baudRate)
{
    m_Serial->setPortName(portName);
    m_Serial->setBaudRate(baudRate);
    m_Serial->setDataBits(QSerialPort::Data8);
    m_Serial->setParity(QSerialPort::NoParity);
    m_Serial->setStopBits(QSerialPort::OneStop);
    m_Serial->setFlowControl(QSerialPort::NoFlowControl);
    // RS485 模式下可能需要设置 RTS 控制
    // m_Serial->setRequestToSend(true);
    if (!m_Serial->open(QIODevice::ReadWrite)) {
        emit Status("语音播报器：" + m_Serial->errorString());
        return false;
    }
    emit Status("语音播报器：串口已打开 " + portName);
    return true;
}

void c_Voice_Remote::Close()
{
    if (m_Serial && m_Serial->isOpen()) m_Serial->close();
}

void c_Voice_Remote::Play(int index)
{
    QByteArray data;
    data.append((index >> 8) & 0xFF);  // 高字节
    data.append(index & 0xFF);         // 低字节
    QByteArray cmd = Build_Cmd(0x5E, data);
    m_Serial->write(cmd);
    emit Status(QString("语音播报器：播放 %1").arg(index));
}

void c_Voice_Remote::Stop()
{
    Play(0);  // index=0 对应停止
}

void c_Voice_Remote::SetVolume(int vol)
{
    if (vol < 1 || vol > 3) return;
    QByteArray data;
    data.append(0x00);
    data.append(vol);
    QByteArray cmd = Build_Cmd(0x2E, data);
    m_Serial->write(cmd);
}

quint8 c_Voice_Remote::Calc_CS(QByteArray &buf)
{
    quint8 cs = 0xFF;
    for (int i = 1; i < buf.size() - 1; i++)
        cs ^= (quint8)buf[i];
    return cs;
}

QByteArray c_Voice_Remote::Build_Cmd(quint8 cmd, QByteArray data)
{
    QByteArray buf;
    buf.append(0xFF);
    buf.append(cmd);
    buf.append(data);
    buf.append(0x00);  // 校验占位
    buf.append(0xAF);
    buf[3] = Calc_CS(buf);
    return buf;
}
```

### 16.9 异常处理

| 异常场景 | 检测方式 | 处理策略 |
|---------|---------|---------|

## 十七、远程 IO 站控制模块（c_IO_Remote / XB6S-MT2002）

> XB6S-MT2002 是华贸(XB6S)系列插片式总线耦合器，通过 Modbus TCP 将远程 IO 模块接入以太网。
> 当前项目配有三个子模块：XB6S-3200（16路 DI）、XB6S-3200B（16路 DO）、XB6S-A04（4路 AO）。

### 17.1 硬件配置

| 模块型号 | 通道数 | Modbus 类型 | 起始地址 | 用途 |
|---------|--------|-----------|---------|------|
| XB6S-MT2002 | — | 总线耦合器 (Profinet→Modbus TCP) | — | 以太网主站 |
| XB6S-3200 | 16 路 | Discrete Inputs | 0 | 传感器/限位开关采集 |
| XB6S-3200B | 16 路 | Coils | 0 | 继电器/执行器控制 |
| XB6S-A04 | 4 路 | Holding Registers | 0 | 比例阀/变频器控制 |

### 17.2 数字量输出接线 (XB6S-3200B)

| 端子 | Coil 地址 | 继电器 | 负载 | 对应方法 |
|------|----------|--------|------|---------|
| Y01 | 0 | KA2 | 左前/左后/右前/右后 三色报警灯(红) | `Set_Alarm_Red()` / `Reset_Alarm_Red()` |
| Y02 | 1 | KA3 | 三色报警灯(黄) | `Set_Alarm_Yellow()` / `Reset_Alarm_Yellow()` |
| Y03 | 2 | KA4 | 三色报警灯(绿) | `Set_Alarm_Green()` / `Reset_Alarm_Green()` |
| Y04 | 3 | KA5 | 散热风扇 | `Set_Fan()` / `Reset_Fan()` |

### 17.3 通信参数

| 项目 | 值 |
|------|-----|
| 协议 | Modbus TCP |
| 默认 IP | 192.168.1.10（可在 UI 配置） |
| 默认端口 | 502 |
| 底层客户端 | `c_ModeBusTCP_Client`（与 CGXi_Remote 共用底层实现） |
| 轮询周期 | 全寄存器类型循环（Coils → HoldingRegisters → DiscreteInputs → InputRegisters） |

### 17.4 数据结构

```cpp
struct s_IO_DB {
    bool Connected = false;                   // 连接状态
    bool Ready = false;                       // 就绪标志（每个轮询周期翻转）
    int FPS = 0;                              // 轮询帧率
    quint16 Coils[1000] = {0};               // 线圈数据缓存
    quint16 DiscreteInputs[1000] = {0};       // 离散输入数据缓存
    quint16 InputRegisters[1000] = {0};       // 输入寄存器数据缓存
    quint16 read_HoldingRegisters[1000] = {0};// 读保持寄存器缓存
    quint16 write_HoldingRegisters[1000] = {0};// 写保持寄存器缓存
};
```

### 17.5 轮询与写入机制

**轮询流程**（与 RGV_Remote 同源模式）：
```
Connect_Done → Read_Coils
  → Read_Coils_Done → memcpy(g_IO.Coils) → Write_Coils_Action/Write_HoldingRegisters_Action
  → Read_HoldingRegisters → Read_HoldingRegisters_Done → memcpy(g_IO.read_HoldingRegisters)
  → Read_DiscreteInputs → Read_DiscreteInputs_Done → FPS 统计 + memcpy(g_IO.DiscreteInputs)
  → Read_InputRegisters → Read_InputRegisters_Done → memcpy(g_IO.InputRegisters)
  → 循环回 Read_Coils
```

**写入验证**（关键设计）：
- `Write_Coils_Action()`：发送写命令，**不立即清除请求标志**
- `Read_Coils_Action()`：回读线圈值后，仅在 `m_Set_Coils[i] && g_IO.Coils[i]` 时确认写成功并清除标志
- `Write_HoldingRegisters_Action()`：同上，发送后不立即清除
- `Read_HoldingRegisters_Action()`：校验 `read == write` 后确认

**可用 API**：
| 方法 | 功能 | 说明 |
|------|------|------|
| `Set_Coil(addr)` / `Reset_Coil(addr)` | 通用线圈置位/复位 | 延迟写入，由轮询自动执行 |
| `Set_Alarm_Red()` / `Reset_Alarm_Red()` | 红色报警灯 | Coil 0 → KA2 |
| `Set_Alarm_Yellow()` / `Reset_Alarm_Yellow()` | 黄色报警灯 | Coil 1 → KA3 |
| `Set_Alarm_Green()` / `Reset_Alarm_Green()` | 绿色报警灯 | Coil 2 → KA4 |
| `Set_Fan()` / `Reset_Fan()` | 散热风扇 | Coil 3 → KA5 |
| `Set_HoldingRegister(addr, value)` | 通用保持寄存器写入 | 延迟写入 |
| `Set_AnalogOutput(ch, value)` | 模拟量输出 | ch 0-3, value 0-65535 |
